# Agent Work Rules for This Project

## Golden Rule
**NEVER change code directly. Always ask the user first.**

## Workflow
1. **Read this file first** — every new session must read AGENTS.md before any work.
2. **Understand the problem fully** — before writing any code, explain what you found and what you plan to change.
3. **Ask permission** — "I found XYZ. Can I fix it by doing ABC?" Wait for user response.
4. **Think about side effects** — every change can break something else. List what could go wrong before coding.
5. **Small changes only** — one fix at a time. No batch refactoring.
6. **Verify with tests** — run relevant tests after every change.
7. **Be honest** — if you're unsure about something, say so. Don't pretend you've fully tested something you haven't.

## Security
- Never commit real secrets, API keys, or credentials.
- Never expose sensitive data in logs or error messages.

## Communication
- User prefers Tamil/English mix. Match their language.
- Be direct and honest. No fluff.
- If you break something, admit it and fix it immediately.
